# 왜 이렇게 바꿨는가

## 1. Python 3.14에서 바로 실행되게 만들기
기존 버전은 FastAPI / Pydantic 기반이라 설치 환경에 따라 `pydantic-core`에서 막힐 수 있었습니다.
그래서 이번 버전은 **표준 라이브러리 중심**으로 바꿔 설치 리스크를 줄였습니다.

## 2. 발표자료 기준으로 재설계
C 압축파일은 참고만 하고, 실제 구조는 발표자료의 흐름인
센서 → 게이트웨이 → 서버/DB → 웹 지도 시각화 기준으로 다시 잡았습니다.

## 3. 왜 Flask/FastAPI가 아니라 http.server 인가
이번 목적은 “설치가 바로 되는 시연용 버전”이므로, 외부 패키지 없이도 동작하는 `http.server`를 선택했습니다.

## 4. 왜 DB를 SQLite로 유지했는가
Python 기본 내장이라 별도 설치가 필요 없고, 발표 시연에서 재현성이 높기 때문입니다.
추후 MariaDB로 옮길 수 있게 테이블 구조는 분리했습니다.

## 5. 왜 판단 로직은 규칙 기반인가
현재는 설명 가능성과 시연 안정성이 중요하므로, 온도/습도/주파수 기반 규칙 점수화를 사용했습니다.
나중에 학습 모델로 교체할 수 있도록 입력 구조를 고정했습니다.
