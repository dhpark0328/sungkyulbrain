# 성결브레인 개인 담당 작업 정리

## 프로젝트 개요

본 프로젝트는 **저가·저전력 도로 결빙 감지 경보 시스템**을 주제로 진행한 전공종합설계 프로젝트이다.

도로 주변에 설치한 센서 노드가 노면 상태와 온습도 데이터를 수집하고, 게이트웨이를 거쳐 서버와 데이터베이스로 전달한 뒤, 웹 화면에서 결빙 위험 상태를 확인할 수 있도록 구성하는 것을 목표로 했다.

기존 고가의 도로 결빙 감지 시스템은 설치 비용과 유지보수 비용이 높아 외곽 도로, 생활 도로, 산간 도로 등에는 적용이 어렵다는 한계가 있다.  
우리 팀은 이러한 문제를 보완하기 위해 저가 센서 노드, LoRa 통신, 서버/데이터베이스, AI 모델, 웹 대시보드 구조를 활용한 경량 시스템을 설계했다.

내가 정리한 주요 담당 범위는 **Server / Database, Front / UI, AWS 서버 환경 구성, 웹서버 이전 준비, 데이터 흐름 정리, Python 시연 코드 정리**이다.

주요 작업은 다음과 같다.

- 서버 및 데이터베이스 구조 정리
- Raspberry Pi 기반 서버 구성 방향 조사 및 정리
- MariaDB / SQLite 등 데이터베이스 사용 방향 비교
- 센서 데이터 저장 구조 설계 정리
- 웹 UI와 서버 데이터 연동 흐름 정리
- 지도 기반 대시보드 구조 정리
- AWS EC2 Ubuntu 서버 오픈 및 접속 확인
- 기존 localhost 웹서버를 AWS 환경으로 옮기기 위한 준비
- DynamoDB, AI 모델, Web 페이지 업데이트 흐름 정리
- Python 기반 시연용 통합 서버 코드 정리
- GitHub 제출용 보안 정리

## 시스템 흐름

전체 시스템 흐름은 다음과 같이 정리할 수 있다.

```text
단말 노드
→ LoRa Gateway
→ DB 저장
→ AI 모델 판단
→ Web 페이지 업데이트
```

단말 노드는 온도, 습도, 노면 상태 관련 데이터를 수집한다.  
게이트웨이는 센서 노드에서 전달된 패킷을 받아 페이로드를 디코딩하고 DB 업로드 로직을 수행한다.  
서버는 저장된 데이터를 AI 모델의 입력값으로 전달하고, 모델 판단 결과를 웹 화면에 반영한다.

## 12주차 진행 자료 반영 내용

추가 진행 자료를 기준으로 다음 내용을 반영했다.

### 하드웨어 단말

- Arduino Pro Mini를 통한 단말 제어
- 전도도에 따른 주파수 변동을 활용한 노면 상태 측정 회로 적용
- DHT22를 통한 실 환경 온습도 측정
- LoRa 기반 저비용·저전력 페이로드 전송 방식 구현
- Sleep mode 제어 적용
- 18650 리튬이온 배터리 2구 사용
- 다중 노드 환경에서 패킷 충돌 방지를 위한 Random Delay 방식 적용

### 게이트웨이

- Raspberry Pi 5 + LoRa HAT 구성
- LoRa 패킷 수신 로직 구현
- 페이로드 디코딩 및 DB 업로드 로직 구현

### 서버 Front

- AWS EC2 Ubuntu 환경 서버 오픈
- Web 서버 구축 및 구동 준비
- UI 보완 예정

### 서버 Back

- AWS EC2 Ubuntu 환경 서버 오픈
- DynamoDB 활용
- 센싱 데이터 DB 저장 후 모델 입력
- AI 모델 결과 반환 후 웹페이지 업데이트 흐름 정리

### AI 모델

- RandomForest 방식 활용
- 더미 데이터 기반 학습
- 추후 실제 데이터 기반 평가 필요
- Python 동작 후 서버 적재 예정

### 진행 예정 사항

- 단말 모듈 소비 전력 측정
- 소비 전력 측정을 통한 기대 동작 수명 확인
- AWS 웹서버 업로드 및 모델 적재
- 단말 → 게이트웨이 → DB → AI → Web 플로우 최종 확인

## 서버 / 데이터베이스 관련 정리

프로젝트 초기에는 Raspberry Pi 기반 서버와 데이터베이스 구성을 검토했다.  
서버에서는 게이트웨이로부터 데이터를 받고, 데이터베이스에 저장한 뒤, 웹에서 조회할 수 있도록 하는 역할을 담당한다.

데이터베이스는 센서 정보, 측정 데이터, 기상 데이터, 추론 결과 등을 분리해서 저장하는 방향으로 정리했다.  
시연용 Python 코드에서는 별도 설치가 쉬운 SQLite를 사용했고, 실제 확장 구조에서는 MariaDB나 DynamoDB와 같은 데이터베이스와 연결될 수 있도록 흐름을 잡았다.

## AWS EC2 서버 정리

웹서버를 로컬 환경에서만 실행하면 외부 접속과 공유가 어렵기 때문에 AWS EC2 서버를 사용했다.

EC2 서버는 Ubuntu 환경으로 구성했고, 개발 및 테스트용으로 사용할 수 있는 인스턴스를 선택했다.  
SSH 접속을 통해 서버가 정상적으로 실행되는지 확인했고, 이후 웹서버를 AWS 환경으로 이전할 수 있는 기반을 마련했다.

보안상 GitHub에는 서버 IP 주소, DNS 주소, SSH 키 파일, AWS API Key 등 민감한 정보는 포함하지 않았다.

## 웹 / UI 관련 정리

웹 화면은 지도 기반 대시보드 형태로 구성했다.  
센서 노드의 위치를 지도에 표시하고, 노면 상태 또는 결빙 위험도에 따라 사용자에게 정보를 제공하는 방향으로 정리했다.

프론트엔드 쪽에서는 HTML, CSS, JavaScript를 활용해 기본 화면 구조를 만들고, 지도 위에 노드와 상태 정보를 표시하는 흐름을 정리했다.

## Python 시연 코드

`src/` 폴더에는 프로젝트 흐름을 시연하기 위한 Python 기반 코드가 포함되어 있다.

이 코드는 실제 하드웨어 전체 연동을 완전히 대체하는 것은 아니며, 다음 흐름을 시연하기 위한 목적이다.

```text
센서 데이터 생성
→ 게이트웨이 전달
→ 서버 저장
→ 노면 상태 판단
→ 웹 지도 화면 표시
```

실행 예시는 다음과 같다.

```bash
cd src
python integrated_app.py
```

브라우저에서 아래 주소로 접속하면 된다.

```text
http://127.0.0.1:8000
```

센서 데이터 시뮬레이션은 다음과 같이 실행할 수 있다.

```bash
python scripts/run_node.py --gateway http://127.0.0.1:8000 --count 8 --mode mixed
```
