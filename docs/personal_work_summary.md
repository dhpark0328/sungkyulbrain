# 개인 담당 작업 상세 정리

## 1. Server / Database

최종보고서의 추진 일정 기준으로 Server/Database 파트는 박경준, 박동현이 담당한 부분이다.  
이 파트에서는 서버 구축 방향, 데이터베이스 사용 방식, 센서 데이터 저장 구조 등을 정리했다.

진행한 내용은 다음과 같다.

- 서버용 Raspberry Pi 설정 및 설계 방향 정리
- 공동 작업을 위한 서버 환경 구성 방향 확인
- Python, Java, Samba 등 작업에 필요한 환경 조사
- MariaDB, MySQL, SQLite 등 데이터베이스 비교
- 프로젝트에 적합한 데이터베이스 구조 검토
- 센서 데이터 저장을 위한 개념적 설계 정리
- 테이블 관계와 기본키, 외래키 구조 정리
- 실제 DB 적용 및 ERD 반환 흐름 정리

## 2. Front / UI

Front/UI 파트도 박경준, 박동현이 함께 담당한 부분이다.  
웹 화면은 사용자가 결빙 위험 상태를 확인할 수 있도록 지도 기반 대시보드 형태로 정리했다.

진행한 내용은 다음과 같다.

- Figma 기반 와이어프레임 구성
- HTML / CSS / JavaScript 개발 환경 구축
- Dashboard 형태의 레이아웃 설계
- 반응형 웹 기초 작업
- 버튼, 카드 등 UI 컴포넌트 스타일 정리
- 지도 연동 및 초기 렌더링 구조 정리
- Polyline을 활용한 경로 표시 로직 정리
- 구간별 데이터에 따른 경로 색상 변경 구조 정리
- 결빙 위험 감지 시 Alert 화면 구현 방향 정리
- 사용자 경험 개선 방향 정리

## 3. AWS EC2 서버

12주차 진행 자료 기준으로 서버 관련 부분에서는 AWS EC2 Ubuntu 환경 서버 오픈, Web 서버 구축 및 구동 준비, DynamoDB, AI 모델 결과 반환, 웹페이지 업데이트 흐름이 정리되어 있다.

내가 정리한 AWS 관련 내용은 다음과 같다.

- AWS EC2 서버 생성
- Ubuntu 운영체제 사용
- 개발 및 테스트용 인스턴스 사용
- SSH 원격 접속 확인
- 서버 기본 상태 확인
- 기존 localhost 웹서버를 AWS로 이전하기 위한 준비
- AWS 웹서버 업로드 및 모델 적재 예정 흐름 정리
- 보안상 민감 정보 제외 후 GitHub 제출용 문서화

## 4. Gateway / DB / AI / Web 흐름

추가 자료의 시스템 구성도와 진행 예정 사항을 기준으로 다음 흐름을 정리했다.

```text
단말
→ 게이트웨이
→ DB
→ AI
→ Web
```

게이트웨이는 Raspberry Pi 5와 LoRa HAT을 이용해 LoRa 패킷을 수신하고, 페이로드를 디코딩한 뒤 DB 업로드 로직을 수행한다.  
서버는 DB에 저장된 센싱 데이터를 AI 모델의 입력값으로 전달하고, AI 모델의 결과를 받아 웹페이지를 업데이트한다.

## 5. Python 시연 코드

업로드한 Python 프로젝트 코드는 단말 → 게이트웨이 → 서버 → DB → 웹 지도 시각화 흐름을 시연하기 위한 코드이다.

구성은 다음과 같다.

- `integrated_app.py`: 통합 서버 실행
- `server_app.py`: 서버 분리 실행용
- `gateway_app.py`: 게이트웨이 분리 실행용
- `scripts/run_node.py`: 센서 노드 시뮬레이션
- `app/db.py`: 데이터베이스 저장 로직
- `app/inference.py`: 노면 상태 판단 로직
- `app/web.py`: 웹 요청 처리
- `static/`: 웹 UI 파일

GitHub 업로드용으로 정리하면서 API Key, API URL, DB 파일, 캐시 파일 등은 제거하거나 환경변수 방식으로 변경했다.
