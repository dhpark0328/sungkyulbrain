# 12주차 진행사항 반영 정리

## 1. 하드웨어 단말 흐름

12주차 진행사항 자료에서는 단말 모듈이 Arduino Pro Mini를 기반으로 구성되었고, 전도도에 따른 주파수 변동을 활용해 노면 상태를 측정하는 구조로 정리되어 있다.

또한 DHT22를 통해 실 환경의 온도와 습도를 측정하고, LoRa를 통해 저비용·저전력 페이로드 전송 방식을 구현하는 방향으로 진행되었다.

단말 관련 주요 내용은 다음과 같다.

- Arduino Pro Mini 기반 단말 제어
- 전도도 기반 노면 상태 측정 회로
- 노면 전극 활용
- DHT22 기반 온습도 측정
- LoRa 기반 페이로드 전송
- Sleep mode 제어 적용
- 18650 리튬이온 배터리 2구 사용
- Random Delay 방식으로 패킷 충돌 방지

## 2. 게이트웨이 흐름

게이트웨이는 Raspberry Pi 5와 LoRa HAT으로 구성되었다.  
LoRa 패킷을 수신하고, 페이로드를 디코딩한 뒤 DB 업로드 로직을 수행하는 역할이다.

주요 내용은 다음과 같다.

- Raspberry Pi 5 + LoRa HAT 구성
- LoRa 패킷 수신 로직 구현
- 페이로드 디코딩
- DB 업로드 로직 구현

## 3. 서버 Front

서버 Front 부분에서는 AWS EC2 Ubuntu 환경 서버 오픈이 완료된 것으로 정리되어 있다.  
이후 Web 서버 구축 및 구동, UI 보완 작업이 진행될 예정인 흐름이다.

주요 내용은 다음과 같다.

- AWS EC2 Ubuntu 환경 서버 오픈
- Web 서버 구축 및 구동 준비
- UI 보완 예정

## 4. 서버 Back

서버 Back 부분에서는 AWS EC2 Ubuntu 환경 서버 오픈, DynamoDB, 센싱 데이터 저장, AI 모델 결과 반환, 웹페이지 업데이트 흐름이 정리되어 있다.

주요 흐름은 다음과 같다.

```text
센싱 데이터
→ DB 저장
→ AI 모델 input
→ AI 모델 결과 반환
→ 웹페이지 업데이트
```

## 5. AI 모델

AI 모델은 RandomForest 방식을 사용하며, 더미 데이터 5000개를 활용해 학습한 것으로 정리되어 있다.  
평가 수치가 높게 나왔지만, 더미 데이터와 단순한 모델 구조로 인한 결과이기 때문에 추후 실제 데이터로 평가가 필요하다고 정리되어 있다.

주요 내용은 다음과 같다.

- RandomForest 방식 활용
- 더미 데이터 5000개 활용
- 안전, 조심, 위험 상태 분류
- 추후 real data 기반 평가 필요
- Python 동작 후 서버 적재 예정

## 6. 챔버 관련

챔버는 팰티어 소재를 활용해 프로토타입을 제작했으나, 냉각 성능 측정 시 기대하던 온도에 도달하지 못한 것으로 정리되어 있다.  
현재는 추가적인 장비 및 시간 투자를 보류하고, 종강 후 전시회 전까지 추가 시도하는 방향이다.

## 7. 기타 진행 예정 사항

추가 진행 예정 사항은 다음과 같다.

- 단말 모듈 소비 전력 측정 시도
- 소비 전력 측정을 통한 기대 동작 수명 확인
- AWS 웹서버 업로드 및 모델 적재
- 단말 → 게이트웨이 → DB → AI → Web 플로우 동작 최종 확인
